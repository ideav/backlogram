# Мультиссылка запросом (пересечение двух списков)

Иногда нужно получить не просто объединение нескольких списков, а только те элементы, которые присутствуют сразу в обоих. Такую выборку называют пересечением списков, или логическим «И»: элемент попадает в результат только тогда, когда он есть и в первом списке, и во втором. В этой статье разберём, как собрать мультиссылку запросом, которая будет именно таким пересечением.

## Введение

Мультиссылка — это ссылка, которая указывает не на один элемент, а сразу на множество элементов. Когда мы строим мультиссылку запросом, мы описываем не конкретный набор элементов, а правило, по которому этот набор вычисляется. Благодаря этому результат остаётся актуальным: как только элемент перестаёт удовлетворять условию, он автоматически исчезает из мультиссылки, а как только начинает удовлетворять — появляется в ней.

Пересечение двух списков — частый случай. Например, у вас есть список «активные клиенты» и список «клиенты из Москвы», а получить нужно тех, кто одновременно и активен, и находится в Москве. Вместо того чтобы вручную сверять оба списка, можно один раз описать запрос, который будет возвращать только общие для обоих списков элементы.

## Предварительные требования

Перед тем как создавать мультиссылку запросом, убедитесь, что:

- у вас есть два списка (две мультиссылки или две именованные выборки), пересечение которых нужно получить;
- вы знаете идентификаторы или имена этих списков, чтобы сослаться на них в запросе;
- вы понимаете разницу между объединением (логическое «ИЛИ», в результат попадают элементы из любого списка) и пересечением (логическое «И», в результат попадают только общие элементы).

Если вы ещё не знакомы с тем, как вообще создаётся мультиссылка запросом, сначала прочитайте статью [Как добавлять мультиссылки запросом](/add-multilink-by-query) — здесь мы опираемся на те же базовые приёмы.

## Пошаговая инструкция

1. Откройте редактор запросов и создайте новую мультиссылку запросом.
2. Сошлитесь на первый список — это будет первое множество элементов.
3. Сошлитесь на второй список — это будет второе множество элементов.
4. Соедините оба списка операцией пересечения (логическим «И»). Именно эта операция оставит только те элементы, которые встречаются и там, и там.
5. Сохраните мультиссылку. Теперь она будет автоматически содержать только пересечение двух списков и обновляться при изменении любого из них.

## Пример

Предположим, у нас есть два списка:

- `список_А` — задачи, помеченные тегом «срочно»;
- `список_Б` — задачи, назначенные на текущего исполнителя.

Чтобы получить только срочные задачи текущего исполнителя, нужно пересечение этих списков:

```
мультиссылка пересечение_срочных:
  список_А И список_Б
```

В результате `пересечение_срочных` будет содержать только те задачи, которые одновременно помечены как срочные и назначены на текущего исполнителя. Если задача перестанет быть срочной или будет переназначена, она автоматически выпадет из мультиссылки.

## Итоги

Пересечение двух списков — это логическое «И»: в результат попадают только элементы, присутствующие сразу в обоих списках. Описав такое правило один раз в виде мультиссылки запросом, вы получаете самообновляющуюся выборку, которую не нужно поддерживать вручную. Это удобно для фильтрации по нескольким условиям одновременно — например, «срочные и мои» или «активные и из нужного региона».
