# Issue 301: «Excel → приложение» — отдельная страница и разбивка на активности

Эта страница фиксирует тематику из [issue #301](https://github.com/ideav/backlogram/issues/301):
сделать так, чтобы Интеграм воспринимался не как «ещё один Low-Code», а как
**сервис, который превращает Excel в работающее веб-приложение быстрее, чем находят фрилансера**.

Документ собран по образцу [`docs/issue-246-blog-design-alternatives.md`](issue-246-blog-design-alternatives.md):
сначала единая тематика, затем разбивка на независимые активности, по каждой из
которых заводится отдельный issue.

## Главная идея

Бизнесмену не нужны квинтет, EAV и `Compile_Report`. Ему нужен один результат:
«Я дал Excel — мне сделали приложение». Поэтому продаём не платформу, а результат
и показываем «магию»: из Excel рождается приложение быстрее, чем варится кофе.

Опорные тезисы тематики:

- **Импортозамещение стало экзистенциальным.** 1С — дорого и медленно; Интеграм —
  быстро и под управлением агента.
- **Усталость от сложности.** «Простая CRM» не должна требовать команды из пяти
  разработчиков, тимлида и девопса.
- **Один понятный угол.** «Замена Excel на приложение за час» — это понятно
  бухгалтеру, логисту, начальнику цеха.

Под капотом уже есть инфраструктура, на которую опираются активности ниже:
форма-CTA на главной (`src/pages/Home.tsx`) уже шлёт заявку в
[`telegram-notify.php`](../public), есть блог (`blog-v2/`) и база знаний
(`src/data/knowledgeBase.ts`) для «историй чуда».

## Независимые активности

Активности самодостаточны и могут выполняться параллельно разными исполнителями.
Каждой соответствует отдельный issue (ссылки добавляются после их создания).

### A1. Лендинг «Загрузите Excel — получите приложение»

Отдельная страница на сайте с формой: пользователь загружает один или несколько
Excel-файлов, указывает тематику и контакт (email / Telegram) — и через ~45 минут
получает ссылку на готовую базу Интеграм со своими данными. Агент работает под
капотом: посетитель не видит `curl` и `_d_new`, видит только результат.

- **Результат:** маршрут вида `/excel-to-app.html` + секция-промо на главной.
- **Критерии готовности:** форма с загрузкой файлов, полем тематики и контактом;
  валидация; экраны успеха/ошибки; мобильная вёрстка; SmartCaptcha как в текущей CTA.

### A2. Бэкенд приёма заявок: файл → issue → Telegram

Серверный обработчик формы из A1 (развитие `telegram-notify.php`): принимает
поля и вложения, создаёт issue в репозитории по токену, прикладывает файлы и
уведомляет Telegram-бот владельца, что появился заказ. Дальше агент запускается
по этому issue вручную.

- **Результат:** PHP-скрипт приёма формы с вложениями.
- **Критерии готовности:** создание issue через GitHub API по токену; вложение
  файлов; уведомление в Telegram; защита от спама (captcha/лимиты); безопасное
  хранение токена (env, не в репозитории).

### A3. Позиционирование «Замена Excel на приложение за час»

Единый оффер на сайте: бьём в одну точку, а не в «Low-Code / квинтетная модель /
EAV». Заголовок и ведущий блок главной переформулируются под понятный угол для
бухгалтера, логиста, начальника цеха.

- **Результат:** обновлённые hero/мета-теги и ключевой блок главной.
- **Критерии готовности:** один явный оффер «из Excel — приложение за час»;
  обновлённые `title`/`description`; призыв к действию ведёт на лендинг из A1.

### A4. Демо-сценарий «Пришлите Excel — получите приложение за 45 минут»

Документ-сценарий для демонстраций и продаж (а не инструкция для разработчиков):
шаги разговора с клиентом и чек-лист действий агента — от получения Excel до
выдачи доступа (`Вход: manager / …`).

- **Результат:** `docs/excel-to-app-demo-playbook.md`.
- **Критерии готовности:** воспроизводимый чек-лист «Excel → доступ»; роли
  (клиент / агент); тайминг ≈45 минут; шаблоны сообщений клиенту.

### A5. Серия «Истории чуда» (кейсы)

Человеческие, а не технические истории успеха. По каждому кейсу — отдельный issue,
прохождение типового пути и описание со скриншотами; публикация в блоге/базе знаний.

Стартовые кейсы из тематики:

- **A5.1.** «Пекарня заменила 5 Excel-файлов на одну систему учёта заказов».
- **A5.2.** «Логистическая компания за день сделала диспетчера отгрузок».
- **A5.3.** «Мастерская по резке рулонов перестала терять заказы».

- **Результат:** по статье на кейс в `blog-v2/src/content/posts/` со скриншотами.
- **Критерии готовности:** на каждый кейс — отдельный issue; пройден типовой путь
  «Excel → приложение»; в статье есть скриншоты результата.
