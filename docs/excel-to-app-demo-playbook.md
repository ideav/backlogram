# Демо-сценарий «Пришлите Excel — получите приложение за 45 минут»

Этот документ — сценарий для **демонстраций и продаж**, а не инструкция для
разработчиков. Он описывает разговор с клиентом и чек-лист действий агента —
от получения Excel-файла до выдачи доступа (`Вход: manager / …`).

Часть тематики [#301](https://github.com/ideav/backlogram/issues/301).
Стратегия и разбивка на активности: [`docs/issue-301-excel-to-app-strategy.md`](issue-301-excel-to-app-strategy.md).

## Когда применять

- Клиент прислал (или готов прислать) Excel-файл и хочет «приложение, а не таблицу».
- Нужна живая демонстрация «магии»: из файла рождается работающее веб-приложение
  быстрее, чем находят фрилансера.
- Продаём **результат**, а не платформу: клиент не видит `curl`, `_d_new`, EAV и
  квинтетную модель — он видит ссылку, логин и свои данные внутри.

## Роли

| Роль | Кто это | Что делает |
| --- | --- | --- |
| **Клиент** | Бухгалтер, логист, начальник цеха, владелец малого бизнеса | Присылает Excel, отвечает на 2–3 вопроса, получает доступ и пробует. |
| **Агент** | Менеджер Интеграма (человек + ИИ под капотом) | Ведёт разговор, превращает Excel в базу, настраивает роли, выдаёт доступ. |

Принцип: вся техническая работа скрыта. Клиент видит только сообщения агента и
итоговую ссылку. Тайминг ниже — ориентир ≈45 минут от файла до входа.

## Тайминг ≈45 минут (обзор)

| Этап | Время | Кто ведёт | Результат этапа |
| --- | --- | --- | --- |
| 0. Контакт и Excel | 0–5 мин | Клиент → Агент | Получен файл и тематика |
| 1. Уточнение | 5–10 мин | Агент | Понятны роли и ключевые поля |
| 2. Сборка базы | 10–30 мин | Агент (под капотом) | Импортированы данные, создана структура |
| 3. Роли и доступ | 30–40 мин | Агент | Логин `manager`, права, ссылка |
| 4. Выдача и показ | 40–45 мин | Агент → Клиент | Клиент вошёл и видит свои данные |

Если данные простые (один лист, понятные колонки) — укладываемся быстрее.
Если файлов несколько или есть связи между листами — этап 2 может занять больше,
тогда честно проговариваем новый ориентир клиенту.

## Воспроизводимый чек-лист «Excel → доступ»

Чек-лист самодостаточен: по нему любой агент проводит демо одинаково.

### Этап 0. Контакт и Excel (0–5 мин)

- [ ] Получить Excel-файл(ы) от клиента (вложение в чат / форму на сайте).
- [ ] Зафиксировать тематику одной фразой: «учёт заказов», «реестр договоров»,
      «склад», «диспетчер отгрузок».
- [ ] Уточнить контакт для выдачи доступа (email или Telegram).
- [ ] Назвать ориентир: «Сделаю рабочее приложение примерно за 45 минут».

### Этап 1. Уточнение (5–10 мин)

- [ ] Открыть файл, бегло понять колонки и что в них лежит.
- [ ] Задать максимум 2–3 вопроса (см. шаблон ниже): кто будет работать с данными
      и что они должны видеть.
- [ ] Определить будущие роли (например: `manager` — полный доступ, `staff` —
      только свои строки).
- [ ] Отметить «грязные» места: объединённые ячейки, несколько таблиц на листе,
      пустые/дублирующиеся заголовки — их чистим на этапе 2.

### Этап 2. Сборка базы (10–30 мин) — под капотом

- [ ] Создать новую базу Интеграма под тематику клиента.
- [ ] Импортировать Excel: первая строка → названия полей, типы определить по данным
      (текст, число, дата, список).
- [ ] Привести структуру в порядок: разбить лист на связанные таблицы, если в нём
      «склеены» сущности (например, «заказы» и «клиенты»).
- [ ] Проверить, что данные на месте: количество строк совпадает, даты и суммы не
      «съехали».
- [ ] Настроить хотя бы один отчёт/представление, понятное клиенту (список заказов,
      реестр договоров).

### Этап 3. Роли и доступ (30–40 мин)

- [ ] Создать пользователя `manager` с временным паролем.
- [ ] Настроить права: что `manager` видит и редактирует.
- [ ] При необходимости добавить вторую роль (`staff`/`viewer`) для показа разграничения.
- [ ] Подготовить ссылку на базу и реквизиты входа в формате `Вход: manager / …`.
- [ ] Самому войти под `manager` и убедиться, что данные открываются.

### Этап 4. Выдача и показ (40–45 мин)

- [ ] Отправить клиенту ссылку и доступ (шаблон «Готово» ниже).
- [ ] Показать 3 вещи: где его данные, как добавить запись, как найти/отфильтровать.
- [ ] Попросить клиента войти самому и добавить одну запись — это «момент магии».
- [ ] Договориться о следующем шаге (доработка полей, ещё роли, перенос остальных
      файлов).

### Контроль готовности (definition of done)

- [ ] Клиент **самостоятельно вошёл** под `manager` и увидел свои данные.
- [ ] Данные из Excel читаются корректно (строки, даты, суммы).
- [ ] Понятно, как добавить и найти запись.
- [ ] Клиент получил ссылку, логин и пароль в одном сообщении.

## Шаблоны сообщений клиенту

Тон — человеческий и короткий. Никакой технической терминологии.

### 0. Приём файла

> Спасибо, файл получил! Чтобы не превратить его обратно в таблицу, а сделать
> удобное приложение, уточню пару моментов — это займёт минуту. Готовое приложение
> пришлю примерно через 45 минут.

### 1. Уточняющие вопросы

> Чтобы настроить всё под вас:
> 1. Кто будет работать с этими данными — вы один или ещё сотрудники?
> 2. Нужно ли, чтобы кто-то видел только свои строки (например, свои заказы)?
> 3. По какому полю чаще всего ищете — по дате, клиенту, номеру?
>
> Если не уверены — отвечайте как есть, остальное настрою по умолчанию.

### 2. Промежуточный статус (по желанию)

> Данные уже внутри, привожу в порядок колонки и настраиваю доступ. Ещё минут
> пятнадцать — и пришлю ссылку для входа.

### 3. Готово — выдача доступа

> Готово! Ваше приложение здесь: <ссылка>
>
> Вход: `manager` / `<временный пароль>`
>
> Внутри уже все ваши данные из Excel. Попробуйте: войдите, найдите любую запись
> и добавьте новую — это занимает пару секунд. Пароль смените на свой в настройках.

### 4. Следующий шаг

> Как ощущения? Если всё нравится — могу перенести остальные файлы, добавить
> сотрудников с их правами и собрать нужные вам отчёты. Что важнее в первую очередь?

## Частые возражения и ответы

| Возражение клиента | Ответ агента |
| --- | --- |
| «А мои данные не потеряются?» | Excel остаётся у вас, мы делаем приложение поверх копии данных. |
| «Это же сложно, нужен айтишник?» | Нет, вы работаете как с обычным сайтом: вход, список, кнопка «добавить». |
| «А если у меня несколько файлов?» | Покажем на одном, остальные перенесём тем же путём — структура уже будет готова. |
| «Сколько это стоит / надолго ли?» | Сначала убедитесь, что удобно, на своих данных — дальше обсудим тариф. |

## Что не делаем в демо

- Не показываем `curl`, `_d_new`, EAV, квинтетную модель и прочую «кухню».
- Не грузим клиента настройками — берём разумные значения по умолчанию.
- Не обещаем перенести все 20 файлов за 45 минут: демо — это один понятный сценарий,
  доведённый до входа.
