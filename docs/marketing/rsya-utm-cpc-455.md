# РСЯ: разметка UTM и цена клика (issue #455)

Кампания **712851090** — «Единая перфоманс-кампания» (ЕПК), 11 групп по числу
посадочных страниц (структура — см. `direct-campaigns-11-pages.md`).

Файл к импорту: **`direct_712851090_filled.xlsx`** — выгрузка с настоящими ID
групп/фраз/объявлений, повторный импорт **обновит** кампанию, а не создаст дубли.

---

## 1. UTM — теперь видно, что это РСЯ

**Было** (одинаково для Поиска и РСЯ — различить в Метрике невозможно):

```
?utm_source=yandex&utm_medium=cpc&utm_campaign=<slug>&utm_content={ad_id}&utm_term={keyword}
```

**Стало** (проставлено во всех 80 строках, колонка «Ссылка»):

```
?utm_source=yandex_{source_type}&utm_medium=cpc&utm_campaign=<slug>&utm_content={ad_id}&utm_term={keyword}&placement={source}
```

Почему через макрос, а не `utm_source=yandex-rsya`: ЕПК крутит Поиск и РСЯ
**в одной кампании**, поэтому зашить источник константой нельзя — его подставляет
Директ на каждом клике.

| Макрос | Что подставляет | Значения |
| :--- | :--- | :--- |
| `{source_type}` | тип площадки | `search` — поиск; `context` — **сети (РСЯ)** |
| `{source}` | место показа | домен площадки в сетях (`travel.ru`); `none` на поиске Яндекса |
| `{ad_id}` | ID объявления | число |
| `{keyword}` | ключевая фраза | текст без минус-слов |

**В отчёте Метрики «Источники → Метки UTM»:**

| utm_source | Это |
| :--- | :--- |
| `yandex_context` | **РСЯ** |
| `yandex_search` | Поиск |

Сайт UTM не парсит — метки читает сам счётчик Метрики (108758829), в коде править
нечего. `placement={source}` — не-UTM параметр, виден в отчёте «Параметры URL»:
показывает, с какого сайта РСЯ пришёл клик (для чистки площадок).

> **Оговорка Яндекса:** `{source}` — не бинарный признак. «При показе на поисковых
> площадках РСЯ может быть передан как домен, так и значение `none`». Поэтому
> тип трафика определяем **только по `{source_type}`**, а `{source}` используем
> как справочную площадку.

## 2. Цена клика 4 ₽ — колонками ставок это НЕ решается

В файле ставки приведены к потолку (12 ₽ → **4 ₽** на 45 строках; автотаргетинг
оставлен на 3 ₽; «Ставка в сетях» проставлена 4 ₽ — раньше была пуста везде).

**Но в ЕПК с показами в РСЯ Директ эти колонки игнорирует.** Официально:

- «В кампаниях с показами в РСЯ или на поиске и РСЯ действуют правила
  GSP-аукциона, ставка устанавливается автоматически, **ручное управление
  недоступно**» — [strategies/manual-strategy](https://yandex.ru/support/direct/ru/strategies/manual-strategy)
- «Назначение ставок работает только для фраз из кампаний с ручным управлением
  **с показами только на поиске**»; «**Ставки в сетях назначаются только
  автоматически**» — [new-interface/bid-wizard](https://yandex.ru/support/direct/ru/new-interface/bid-wizard)

То есть ставки в XLSX — это фиксация намерения; они заработают, только если
кампания станет поисковой с ручными ставками.

### Что реально ограничивает цену клика

| Хочу | Как | Это потолок? |
| :--- | :--- | :--- |
| **РСЯ, ~4 ₽** | ЕПК с местами показа **только РСЯ** → стратегия «Максимум кликов» → **средняя цена клика = 4 ₽** | ❌ **Нет.** Это недельное **среднее**: отдельные клики уходят выше. При <100 кликов/неделю среднее и само может превысить лимит **до 2×** |
| **Жёсткие 4 ₽** | ЕПК **только Поиск** → «Максимум кликов **с ручными ставками**» → ставка 4 ₽ | ✅ Да, списываемая цена ≤ ставки. Но это **не РСЯ** |

Максимальной цены клика в ЕПК с РСЯ **не существует** — стратегия «Максимум
кликов» предлагает только два ограничителя: средняя цена клика (мин. 0,9 ₽) или
недельный бюджет (мин. 300 ₽) — [strategies/average-cpc](https://yandex.ru/support/direct/ru/strategies/average-cpc).

> Настройка «ограничить максимальную ставку» в документации существует, но она —
> в стратегии **«Продвижение приложений»**, не в ЕПК. Не перепутать.

### Порядок действий в кабинете

Файлом это не импортируется — места показа и стратегия живут на уровне кампании:

1. Кампания 712851090 → **Места показа** → снять всё, кроме **«Рекламная сеть
   Яндекса»** (шаг 1 создания/редактирования кампании).
2. **Стратегия** → «Максимум кликов» → ограничение **«Средняя цена клика» = 4 ₽**.
3. Импортировать `direct_712851090_filled.xlsx` (Коммандер → кампания → «Импорт
   из файла») — подтянет новые UTM.

> **Гоча:** переключение мест показа с сетей на поиск/карты **авто-включает
> автотаргетинг** во всех группах по всем категориям запросов. После любой
> правки мест показа — проверить автотаргетинг.

Разделить бюджет/ставки между Поиском и РСЯ **внутри одной ЕПК нельзя** —
стратегия и лимиты общие на кампанию. Нужна разная экономика → нужны **разные
кампании** (ЕПК-поиск / ЕПК-РСЯ).

## 4. Блокер: в кампании нет ни одной картинки

Колонки «Изображение 1…5», «Изображение», «Креатив» — **пусты во всех 80
строках**. РСЯ — это графические места показа: без картинок/креативов объявления
в сетях практически не показываются. Ограничение цены клика не поможет, если
показов нет.

Вынесено отдельной задачей (**issue #457**) — картинки требуют загрузки
креативов в кабинет (в XLSX едут хеши/URL уже загруженных изображений).
