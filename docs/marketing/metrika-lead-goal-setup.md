# Цель Метрики «lead» — настройка и выбор в Директе

Чтобы реклама оптимизировалась на **заявки**, а не просто на клики, нужна цель
конверсии в Яндекс.Метрике. В коде это уже сделано; остался один разовый шаг в
интерфейсе Метрики.

## Что за цель

При **успешной отправке любой формы-заявки** сайт вызывает:

```js
ym(108758829, 'reachGoal', 'lead')
```

— отправляет в счётчик Метрики **108758829** событие с идентификатором **`lead`**
(см. `src/lib/metrika.ts`, подключено во всех формах). Чтобы Метрика считала это
конверсией, а Директ мог на неё оптимизироваться, нужно создать цель типа
**«JavaScript-событие»** с тем же идентификатором `lead`.

## Шаг 1. Создать цель в Метрике

1. **metrika.yandex.ru** → выбрать счётчик **108758829**.
2. Слева **Настройка** → вкладка **«Цели»** → **«Добавить цель»**.
3. Заполнить:
   - **Название цели:** `Заявка (lead)` (любое, для себя).
   - **Тип условия:** **«JavaScript-событие»**.
   - **Идентификатор цели:** `lead` — ровно так, маленькими буквами, без пробелов
     (должно **буква-в-букву** совпадать со строкой в коде).
4. **«Добавить цель»** → сохранить.

Заявки со всех страниц пойдут в одну цель `lead`; в отчётах их можно разделить
по параметру визита `source` (`konstruktor-prilozhenij`, `uc-sklad`, `uc-crm` …),
который передаётся вместе с целью.

## Шаг 2. Проверить (после деплоя)

- Отправить тестовую заявку через форму на сайте.
- Метрика → **Отчёты → Конверсии** — через несколько минут появится конверсия по
  цели `lead` (или смотреть в Вебвизоре / реальном времени).
- Технически: в консоли браузера при отправке уходит
  `ym(108758829,'reachGoal','lead', {source})`.

## Шаг 3. Выбрать цель в Директе

1. **Параметры кампании** → раздел **«Счётчики Метрики»** — убедиться, что
   привязан счётчик **108758829** (без этого Директ не видит цели).
2. В стратегии **«Максимум конверсий»** в поле **«Целевое действие» / «Ключевая
   цель»** выбрать **`Заявка (lead)`**.
3. Сохранить.

До накопления ~10 конверсий держать **«Максимум кликов»**, затем переключиться на
**«Максимум конверсий»** по цели `lead`.

## Частые ошибки

- **Счётчик и Директ не связаны** — цель не появится в списке. Нужен один логин
  Яндекса или выданный доступ к счётчику 108758829.
- **Идентификатор не совпадает** — если в Метрике написать `Lead`, `lead ` (с
  пробелом) или другое — события не свяжутся с целью. Только `lead`.
- **Смотрят статистику сразу** — конверсии появляются с задержкой в несколько
  минут; исторические (до создания цели) по JS-событию могут не подтянуться.

Связано: цель `lead` уже прописана в UTM материалов кампаний
(`direct-campaigns-11-pages.md`, `yandex-direct-excel-constructor.md`).
